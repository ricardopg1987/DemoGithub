Comandos y Preguntas Frecuentes de Git y GitHub

Comandos Básicos de Git

  • git init — Inicializa un nuevo repositorio Git.
  • git clone <url> — Clona un repositorio remoto.
  • git status — Muestra el estado de los archivos.
  • git add <archivo> — Añade archivos al área de staging.
  • git commit -m "mensaje" — Guarda los cambios en el repositorio.
  • git push — Envía los cambios al repositorio remoto.
  • git pull — Descarga y fusiona cambios del remoto.
  • git branch — Lista las ramas existentes.
  • git checkout <rama> — Cambia de rama.
  • git merge <rama> — Fusiona una rama con la actual.
  • git log — Muestra el historial de commits.
  • git remote -v — Lista los repositorios remotos.

Preguntas Frecuentes

  • ¿Qué es Git?
    Un sistema de control de versiones distribuido.
  • ¿Qué es GitHub?
    Una plataforma para alojar repositorios Git y colaborar en proyectos.
  • ¿Cómo subo un proyecto a GitHub?
    Inicializa el repositorio con git init, haz commit de tus archivos, crea un repositorio en GitHub y usa git remote add origin <url> y git push -u origin main.
  • ¿Cómo soluciono conflictos de merge?
    Edita los archivos en conflicto, resuélvelos y haz git add seguido de git commit.
  • ¿Cómo ignoro archivos?
    Agrega los nombres de los archivos o carpetas en un archivo .gitignore.
  • ¿Cómo elimino un archivo del repositorio?
    Usa git rm <archivo> y luego git commit.
  • ¿Cómo veo los cambios realizados?
    Usa git log para ver el historial y git diff para ver diferencias.
